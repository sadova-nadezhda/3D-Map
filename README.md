# Интерактивная карта с фургоном

## О проекте

Это интерактивная 3D-карта на Three.js, где фургон перемещается между городами по заданному маршруту.

Текущая логика проекта:

* модалка открывается только по клику на активный город
* если выбран следующий доступный город, фургон сначала доезжает до него, и только потом открывается модалка
* маршруты открываются в определённом порядке
* во время движения фургона клики по городам должны быть заблокированы

## Назначение файлов

### `src/main.js`

Главная точка входа.

Здесь:

* собираются все модули
* создаются `sceneContext`, `state`, `modal`, `labels`, `route`, `van`
* хранится `selectCity()`
* обрабатывается клик по карте
* запускается `animate()`
* вызывается `loadScene()`

Открывай этот файл, если нужно:

* поменять общую логику приложения
* изменить поведение выбора города
* изменить связку между модулями
* добавить глобальные обработчики событий

---

Здесь хранятся:

* `CITY_CONTENT` — названия и описания городов
* `ROUTE_ORDER` — порядок доступных маршрутов

Открывай этот файл, если нужно:

* поменять тексты в модалках
* переименовать города
* изменить порядок маршрута

## Поток работы приложения

### 1. Запуск

`main.js` создаёт сцену, состояние и контроллеры.

### 2. Загрузка сцены

`loadScene.js` загружает карту, фургон, свет и города.

### 3. Выбор города

`main.js` обрабатывает клик и вызывает `selectCity()`.

### 4. Построение маршрута

`route.js` строит кривую и отрисовывает дорогу.

### 5. Движение фургона

`van.js` двигает фургон по маршруту.

### 6. UI

`labels.js` обновляет подписи, `modal.js` показывает модалку.

## Рекомендации по поддержке

Чтобы проект не разрастался хаотично:

* не класть большую логику маршрута в `main.js`
* не класть тексты городов в `main.js`
* не смешивать UI и Three.js код в одном модуле
* всё общее состояние хранить в `state.js`
* всё, что относится к пути, держать в `route.js`
* всё, что относится к фургону, держать в `van.js`
